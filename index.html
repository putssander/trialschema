Expected source format

TrialSchema accepts four formats and auto-detects on upload:

  • Excel / CSV (Dutch corpus layout): columns Naam studie, inclusion_criteria, exclusion_criteria, etc.
  • TrialGPT JSONL: one trial per line with _id, title, metadata.inclusion_criteria.
  • Org-specific JSON/JSONL: array of objects with trial_id, criteria, care_path.

Not sure? Download an empty template from the sidebar, fill it in with one trial, and re-upload.

Higher match accuracy

Every criterion is rewritten to be unambiguous and self-contained — no more guessing what "locally advanced" or "HER2-low" meant.

via per-criterion Clarify rewrites

Look in the right place

Each rule is pointed at the patient documents where the answer lives — pathology, labs, imaging, MDT notes — so your agent or matching system stops searching everywhere.

via the doc-routing chips per criterion

Fail fast, save compute

Cheap deterministic checks (age, lab values) run first; expensive LLM reasoning only fires for patients still in the funnel.

via easy→hard criterion ordering
