Expected source format

TrialSchema accepts these formats and auto-detects on upload:

  • Excel / CSV: required trial and at least one of inclusion / exclusion.
  • Optional spreadsheet columns: trial_id, care_path, active, condition, indication, start_date, completion_date.
  • Header normalization: spaces, hyphens, and underscores are treated the same, so care path maps to care_path.
  • TrialGPT JSONL: one trial per line with _id, title, metadata.inclusion_criteria.
  • Org-specific JSON/JSONL: array of objects with trial_id, criteria, care_path.

Not sure? Download an empty template from the sidebar, fill it in with one trial, and re-upload.

Microsoft 365 Copilot token mode

Token paste is a temporary testing path. A Microsoft Graph access token is short-lived and belongs to the signed-in work account. For normal end users, use a future Sign in with Microsoft flow instead of asking them to copy tokens.

Only paste a token on trusted TrialSchema pages. The token is stored in this browser's localStorage and expires automatically.
  1. Open Microsoft Graph Explorer and sign in with your work account.
  2. Switch the endpoint version to beta, choose POST, and run /copilot/conversations with body {}.
  3. Copy the returned conversation id, then run /copilot/conversations/{id}/chat with body {"message":{"text":"Reply with OK."},"locationHint":{"timeZone":"Europe/Amsterdam"}}.
  4. If prompted, use Modify permissions or Consent to permissions. Your tenant admin may need to approve all Copilot Chat API delegated scopes.
  5. If Graph Explorer exposes an Access token panel/menu, copy that bearer token after both calls work and paste it into TrialSchema's Copilot token field.
  6. If no token copy option is available, use Postman or wait for the built-in Microsoft sign-in flow; end users should not need this manual step.

The token must target Microsoft Graph, belong to a work account with a Microsoft 365 Copilot add-on license, and include the delegated scopes Sites.Read.All, Mail.Read, People.Read.All, OnlineMeetingTranscript.Read.All, Chat.Read, ChannelMessage.Read.All, and ExternalItem.Read.All. If creating a conversation works but chat returns 403, the conversation-create check was not enough; verify the same /chat request in Graph Explorer. Tenant consent, the Copilot add-on license, or API entitlement may still be missing. TrialSchema sends prompt text only; it does not attach OneDrive or SharePoint files.

If the chat response says you don't have a valid license, Graph permissions are not the blocker. The signed-in work account needs the Microsoft 365 Copilot add-on license assigned or activated, followed by a fresh token.

If Graph Explorer returns UnknownError on /copilot/conversations, save the response request id and date for tenant admin or Microsoft support. That failure happens before TrialSchema sends any trial prompt.

Document Types & Visit Order

Where agents should look in the patient record.

Defines the patient-document types available to the extractor and matcher, the default scan set, and the general visit order. Criteria can still route to specific primary and fallback documents when a rule clearly belongs somewhere else.

Care Paths

Normalized care paths for first-pass matching.

Normalizes source care-path values into stable English care-path ids, such as breast cancer or heart failure. Matching agents can use these as a first-pass filter before evaluating detailed eligibility criteria; a trial can belong to several care paths.
